# IT Services Complaints & Requests Portal

This is a comprehensive ASP.NET WebForms application (C# 6, .NET Framework 4.6, Visual Studio 2015) designed for managing IT Service Requests and Complaints.

## Project Structure
- **ComplaintsPortal.Web**: The ASP.NET WebForms front-end application containing Master Pages, Employee UI, Admin UI, and Tech Expert pools.
- **ComplaintsPortal.BusinessLogic**: BLL managers that encapsulate workflow rules, active directory integration, and entity tracking.
- **ComplaintsPortal.DataAccess**: DAL using pure ADO.NET (`Oracle.ManagedDataAccess`) for communicating with an Oracle 11g database.
- **ComplaintsPortal.Entities**: POCO classes defining the schema models.

## Database Setup (Oracle 11g)

The system relies on a set of core tables, some of which were added during Phase 1-2, and others added during Phase 3. The `hrdata.empdetails` table is assumed to already exist in your environment.

### Core Master & Transaction Tables (Phase 1 & 2)
```sql
-- Divisions and Roles
CREATE TABLE MST_DIVISION (DIVISION_ID NUMBER PRIMARY KEY, DIVISION_CODE VARCHAR2(50), DIVISION_NAME VARCHAR2(100), IS_ACTIVE VARCHAR2(1) DEFAULT 'Y');
CREATE TABLE MST_ROLE (ROLE_ID NUMBER PRIMARY KEY, ROLE_CODE VARCHAR2(50), ROLE_NAME VARCHAR2(100), ROLE_CATEGORY VARCHAR2(50), REQUIRES_DIVISION VARCHAR2(1), IS_ACTIVE VARCHAR2(1) DEFAULT 'Y');
CREATE TABLE TRN_USER_ROLE (USER_ROLE_ID NUMBER PRIMARY KEY, PCNO VARCHAR2(20), ROLE_ID NUMBER, DIVISION_ID NUMBER, EFFECTIVE_FROM DATE, EFFECTIVE_TO DATE, IS_ACTIVE VARCHAR2(1) DEFAULT 'Y', CREATED_BY VARCHAR2(20), CREATED_DATE TIMESTAMP);

-- Services and Request Types
CREATE TABLE MST_SERVICE (SERVICE_ID NUMBER PRIMARY KEY, SERVICE_CODE VARCHAR2(50), SERVICE_NAME VARCHAR2(100), IS_ACTIVE VARCHAR2(1) DEFAULT 'Y');
CREATE TABLE MST_REQUEST_TYPE (REQUEST_TYPE_ID NUMBER PRIMARY KEY, SERVICE_ID NUMBER, TYPE_CODE VARCHAR2(50), TYPE_NAME VARCHAR2(100), IS_FLOW_BASED VARCHAR2(1), IS_ACTIVE VARCHAR2(1) DEFAULT 'Y');
CREATE TABLE MST_REQUEST_SUBTYPE (SUBTYPE_ID NUMBER PRIMARY KEY, REQUEST_TYPE_ID NUMBER, SUBTYPE_CODE VARCHAR2(50), SUBTYPE_NAME VARCHAR2(100), IS_ACTIVE VARCHAR2(1) DEFAULT 'Y');

-- Tech Expert Mapping
CREATE TABLE TRN_TECH_EXPERT_SERVICE (MAPPING_ID NUMBER PRIMARY KEY, PCNO VARCHAR2(20), SERVICE_ID NUMBER, REQUEST_TYPE_ID NUMBER, ASSIGNED_BY VARCHAR2(20), ASSIGNED_DATE TIMESTAMP, IS_ACTIVE VARCHAR2(1) DEFAULT 'Y');

-- Standby Assignment
CREATE TABLE TRN_STANDBY_ASSIGNMENT (STANDBY_ID NUMBER PRIMARY KEY, FOR_USER_ROLE_ID NUMBER, STANDBY_PCNO VARCHAR2(20), START_DATE TIMESTAMP, END_DATE TIMESTAMP, ASSIGNED_BY VARCHAR2(20), ASSIGNED_DATE TIMESTAMP, IS_ACTIVE VARCHAR2(1));

-- Workflow Engine
CREATE TABLE MST_WORKFLOW (WORKFLOW_ID NUMBER PRIMARY KEY, REQUEST_TYPE_ID NUMBER, SUBTYPE_ID NUMBER, WORKFLOW_NAME VARCHAR2(100), IS_ACTIVE VARCHAR2(1) DEFAULT 'Y', CREATED_BY VARCHAR2(20), CREATED_DATE TIMESTAMP);
CREATE TABLE MST_WORKFLOW_STAGE (STAGE_ID NUMBER PRIMARY KEY, WORKFLOW_ID NUMBER, STAGE_SEQ NUMBER, STAGE_NAME VARCHAR2(100), APPROVER_ROLE_ID NUMBER, ASSIGNMENT_MODE VARCHAR2(50), REJECT_TARGET VARCHAR2(50));

-- Core Requests
CREATE TABLE TRN_REQUEST (
    REQUEST_ID NUMBER PRIMARY KEY, 
    REQUEST_NUMBER VARCHAR2(50), 
    REQUEST_TYPE_ID NUMBER, 
    SUBTYPE_ID NUMBER, 
    WORKFLOW_ID NUMBER, 
    REQUESTER_PCNO VARCHAR2(20), 
    DIVISION_ID NUMBER, 
    CURRENT_STAGE_ID NUMBER, 
    STATUS VARCHAR2(50), 
    PICKED_BY_PCNO VARCHAR2(20), 
    PICKED_DATE TIMESTAMP, 
    REJECTION_COUNT NUMBER DEFAULT 0, 
    SUBMITTED_DATE TIMESTAMP, 
    CLOSED_DATE TIMESTAMP,
    ROOM_NO VARCHAR2(50), BUILDING VARCHAR2(100), FLOOR VARCHAR2(50), DESCRIPTION CLOB, RESOLUTION_REMARKS CLOB
);
CREATE TABLE TRN_REQUEST_STAGE_HISTORY (HISTORY_ID NUMBER PRIMARY KEY, REQUEST_ID NUMBER, STAGE_ID NUMBER, ACTION_BY_PCNO VARCHAR2(20), ACTED_AS_STANDBY NUMBER(1), STANDBY_FOR_USER_ROLE_ID NUMBER, ACTION VARCHAR2(50), REMARKS CLOB, ACTION_DATE TIMESTAMP);

-- Auditing
CREATE TABLE TRN_AUDIT_LOG (AUDIT_ID NUMBER PRIMARY KEY, PCNO VARCHAR2(20), MODULE_NAME VARCHAR2(50), ENTITY_ID VARCHAR2(50), ACTION_TYPE VARCHAR2(50), ACTION_DESC VARCHAR2(500), IP_ADDRESS VARCHAR2(50), ACTION_DATE TIMESTAMP);

-- Sequences
CREATE SEQUENCE SEQ_REQUEST START WITH 1 INCREMENT BY 1;
```

### Phase 3 Tables (Dynamic Fields & Connection Register)
Run these commands to prepare your Oracle 11g database for Phase 3 features:
```sql
-- Dynamic Field Definitions (Configured by Admin)
CREATE TABLE MST_FIELD_DEFINITION (
    FIELD_ID NUMBER PRIMARY KEY,
    REQUEST_TYPE_ID NUMBER NOT NULL,
    SUBTYPE_ID NUMBER,
    FIELD_CODE VARCHAR2(50) NOT NULL,
    FIELD_LABEL VARCHAR2(150) NOT NULL,
    FIELD_TYPE VARCHAR2(20) NOT NULL,  -- e.g., TEXT, MULTILINE, NUMBER, DATE, DROPDOWN
    DROPDOWN_OPTIONS VARCHAR2(500),    -- Comma-separated options if type is DROPDOWN
    IS_MANDATORY VARCHAR2(1) DEFAULT 'N',
    DISPLAY_ORDER NUMBER DEFAULT 10,
    IS_ACTIVE VARCHAR2(1) DEFAULT 'Y'
);

-- Captured Form Values for Dynamic Fields
CREATE TABLE TRN_REQUEST_FIELD_VALUE (
    VALUE_ID NUMBER PRIMARY KEY,
    REQUEST_ID NUMBER NOT NULL,
    FIELD_ID NUMBER NOT NULL,
    VALUE_TEXT CLOB,
    VALUE_NUMBER NUMBER,
    VALUE_DATE TIMESTAMP
);

-- Active Connection Register (Managed by Tech Experts)
-- Note: The application attempts to auto-create this table if it detects it is missing, 
-- but it is recommended to create it manually using the script below.
CREATE TABLE TRN_CONNECTION_REGISTER (
    CONN_ID NUMBER PRIMARY KEY,
    PCNO VARCHAR2(20) NOT NULL,
    CONNECTION_TYPE VARCHAR2(50) NOT NULL,
    IP_ADDRESS VARCHAR2(50),
    MAC_ADDRESS VARCHAR2(50),
    PORT_NO VARCHAR2(50),
    SWITCH_NAME VARCHAR2(100),
    ASSIGNED_DATE TIMESTAMP DEFAULT SYSTIMESTAMP,
    STATUS VARCHAR2(20) DEFAULT 'ACTIVE'
);
```

## Compilation and Build
- Ensure you have **Visual Studio 2015** (or newer).
- Make sure **.NET Framework 4.6** Developer Pack is installed on your system.
- The project is fully compatible with C# 6.0 standards.
- Build the solution using Visual Studio or MSBuild.
